# Frontend Package Version Display

## Goal

Show the published Canton Explorer package version in the frontend footer next
to the existing SDK attribution.

## Design

The frontend build will read the version from `backend/package.json` and expose
it as a Vite build-time constant. `App.vue` will render that constant as
`version <value>` in the existing footer. This keeps the displayed version
aligned with the npm package without adding a runtime API request.

The approach works for both Vite development builds and the packaged frontend
generated by the backend prepack script. The backend package remains the single
source of truth; `frontend/package.json` will not receive a second version.

## Testing

Add an `App.vue` regression assertion that the footer contains the configured
package version. Run the focused frontend test and the frontend production
build to verify the constant is available in both the test and Vite build
pipelines.

## Scope

No backend endpoint, runtime fetch, version negotiation, or new settings UI is
needed.
